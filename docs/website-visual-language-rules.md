## 1. 這份文件的定位

前一份文件定的是設計原則。
這一份開始把原則變成真正可落地的視覺規則。

目標不是一次決定所有細節，而是先建立 VELOCUE 官網的共同語法：

- 色彩怎麼分工
- 字體怎麼分層
- 留白怎麼控制
- section 怎麼長
- 元件該有什麼氣質

這樣後面不管是做首頁、補次頁、還是重做產品介紹，都不需要每次重新談審美。

## 2. 視覺系統一句話

VELOCUE 的網站視覺語言應該是：

`日光感的留白編排，加上成熟克制的工具結構`

翻成實作規則就是：

- 有空氣感
- 有暖度
- 有大標識別
- 有工具秩序
- 沒有 dashboard 壓迫感

## 3. 色彩系統規則

這一版先定色彩角色與比例，不先鎖最終色碼。

### 3.1 色彩角色

#### A. Base Canvas

網站的大底色。
用途是建立整體氣氛與留白質地。

規則：

- 使用暖白、米白、淡奶油白、偏陽光灰白
- 避免純冷白整頁鋪滿
- 避免過黃，否則容易變成餐飲或 lifestyle 感

定位：

- 有光感
- 不搶內容
- 能讓深色文字穩定站住

#### B. Surface

卡片、區塊容器、局部底色。

規則：

- 比 Base Canvas 稍微更白或更有層次
- 與背景的差異要靠明度，不靠重裝飾
- 可在不同 section 中使用 2 到 3 種細微明度差建立節奏

#### C. Ink

主文字與主要結構色。

規則：

- 不用純黑
- 偏深棕灰、深石墨灰、暖深灰
- 用來建立穩定成熟感

#### D. Accent Primary

主要行動色，用於主 CTA、重點連結、小範圍關鍵高亮。

規則：

- 應乾淨、有自信、易辨識
- 不必等於暖陽主色
- 可偏沉穩藍、深青藍、墨綠藍一類較穩的方向
- 避免過度夢幻或糖果亮色

#### E. Accent Warm

品牌情緒點綴色。
用於背景光感、裝飾形體、重點區塊氣氛。

規則：

- 偏日照橙、淡杏色、柔暖金、曬過太陽的沙色
- 不作為全站功能色主角
- 不直接拿來大量上按鈕

#### F. Support Neutral

邊框、說明文字、輔助資訊、次級按鈕。

規則：

- 需足夠穩定
- 不可偏藍灰過重
- 不可過淡到失去層次

### 3.2 配色比例建議

首頁畫面整體比例建議：

- 60% Base Canvas / 大面積留白底
- 20% Surface / 區塊容器與局部背景
- 10% Ink / 文字與主要結構
- 5% Accent Warm / 情緒點綴
- 5% Accent Primary / 行動與關鍵引導

規則：

- 品牌暖色永遠不能比文字和結構更吵
- CTA 色的存在感要準，但不能滿頁重複
- 若某一頁開始像在展示色票，代表配色已經過量

### 3.3 色彩禁忌

- 不要全頁只靠米色和白色撐版面
- 不要讓暖色同時主導背景、按鈕、插圖、標題
- 不要把產品圖表色直接搬來當首頁主視覺色
- 不要使用過多發光漸層與玻璃感
- 不要為了活潑而引入太多高彩度顏色

## 4. 字體系統規則

### 4.1 字體角色

#### A. Display Typeface

用途：

- Hero 主標
- 大 section 標題
- 短句型品牌主張

規則：

- 圓潤、厚實、現代
- 有份量，但不可愛
- 適合短句，不適合長段落
- 應有明顯品牌辨識性

判斷標準：

- 看起來像成熟的 rounded display，而不是兒童向圓體
- 在中文粗體下仍然有結構
- 與工具產品並排時不會顯得太鬆

#### B. Text Typeface

用途：

- 內文
- 功能說明
- 卡片文案
- 次標

規則：

- 清晰、穩定、低個性
- 不要與 display 同樣圓
- 能支撐中等篇幅閱讀

#### C. UI / Numeric Typeface

用途：

- 數字
- 小標籤
- 按鈕
- 路線距離、海拔、功能資訊

規則：

- 與 text 字體協調
- 數字辨識要強
- 單位與數值並排時要俐落

### 4.2 字級層級建議

首頁至少要有以下層級：

- Display XL：Hero 主標
- Display L：section 主標
- Heading M：內容區標題
- Body L：副標與重點說明
- Body M：一般段落
- Meta / Label：小標籤與補充資訊

規則：

- Hero 主標的存在感要明顯，但只保留在少數位置
- 同一頁不要出現太多「很大又很粗」的標題互搶
- 小字不是用來塞資訊，而是用來補結構

### 4.3 中文排版規則

- 行高寬鬆，但不鬆散
- 段落寬度要控制，避免橫向太長
- Hero 文案不宜寫成過長完整句
- 副標可稍長，但應保持自然停頓
- 數字、公里、百分比等資訊要注意字重平衡

## 5. 間距系統規則

### 5.1 間距哲學

VELOCUE 的留白不是裝飾，而是讓閱讀和內容排序更從容。

因此間距應該做到：

- section 與 section 明顯分段
- 區塊內部又足夠緊密，不會散掉
- 讓頁面有呼吸，但不顯得內容稀薄

### 5.2 節奏規則

- section 間距要大於一般網站常見模板
- 標題與副標之間距離應緊於 section 與 section 之間
- 卡片內部間距應有重量感，避免擁擠
- CTA 與說明文字之間需要明確分組

### 5.3 間距判斷方法

如果一個畫面看起來：

- 像 dashboard：間距太小
- 像海報模板：可能只有大留白沒有資訊秩序
- 像簡報：區塊切分太硬

VELOCUE 應介於：

`品牌首頁的呼吸感` 和 `工具頁的秩序感` 之間

## 6. 圓角、邊框、陰影規則

### 6.1 圓角

規則：

- 使用偏大但一致的圓角
- 大容器可圓，小元件則節制
- 同頁不要出現過多不同圓角語言

目標：

- 友善
- 現代
- 有溫度

避免：

- Q 彈感
- 泡泡感
- 每個元素都非常圓

### 6.2 邊框

規則：

- 使用細、低對比、穩定的邊框
- 邊框主要是整理結構，不是強調存在
- 能不用邊框時，可用背景明度差或間距代替

### 6.3 陰影

規則：

- 陰影要薄、柔、低高度
- 不使用厚重漂浮卡片
- 不用 startup landing page 常見霓虹光暈

陰影目的：

- 幫助容器從背景浮出一點點
- 不是製造誇張視覺特效

## 7. 首頁 section 模板規則

### 7.1 Hero 模板

組成：

- 左文右圖，或上文下圖
- 主標一句
- 副標一小段
- 主要 CTA 一個
- 次 CTA 一個
- 產品主視覺一組

規則：

- Hero 文案區寬度需受控
- 產品圖需帶一點編排感，不是直接 screenshot 貼上
- Hero 的品牌感要比功能細節更強
- 但一定要看得出是工具，不可純抽象

### 7.2 Proof Block 模板

組成：

- 一個主論點
- 兩到四個具體情境或能力
- 一張核心產品示意

規則：

- 每段 proof block 只證明一個核心說法
- 圖與文需互相補充，不可重複
- 可以用簡化圖像取代完整 UI 截圖

### 7.3 Method Block 模板

組成：

- 3 到 4 個步驟
- 每步搭配一句說明
- 可有簡潔圖示或小型介面片段

規則：

- 不要像產品 onboarding
- 重點在讓人理解方法，不是教學細節

### 7.4 Result Block 模板

組成：

- 一個總結句
- 2 到 3 個結果點
- 可搭配小型文字組或視覺點綴

規則：

- 結果語句要回到騎前準備
- 不直接承諾成績
- 不要寫得像運動品牌 slogan

## 8. 元件語言規則

### 8.1 Button

主按鈕規則：

- 寬厚但不笨重
- 圓角清楚
- 字重穩
- 顏色集中在主 CTA

次按鈕規則：

- 可以更淡、更像引導而非促銷
- 適合搭配 outline 或低對比 surface

避免：

- 過亮色塊
- 太多 hover 戲劇效果
- 促銷式強對比按鈕堆疊

### 8.2 Card

規則：

- 不是所有內容都要卡片化
- 卡片只用在需要承接內容分組時
- 卡片內文要短、穩、清楚
- 一頁卡片樣式不宜超過 2 到 3 種

### 8.3 Badge / Label

規則：

- 以俐落、簡潔為主
- 不要做成可愛貼紙感
- 若用於 section 小分類，應像編輯註記，不像 UI 狀態燈

### 8.4 Screenshot Frame

規則：

- 可以有柔和外框或容器感
- 不要模擬太重的裝置外殼
- 不要讓產品畫面被過度裝飾埋掉

## 9. 產品畫面呈現規則

VELOCUE 首頁很依賴產品證據，所以產品畫面處理很重要。

### 9.1 畫面選擇規則

- 一張圖只服務一個敘事點
- 優先展示路線、等高線、註記、輸出之間的因果關係
- 不選資訊太滿或狀態太雜的畫面

### 9.2 畫面編排規則

- 可局部放大關鍵區域
- 可用層疊方式表現流程
- 可搭配文字註解，但不可變成規格圖解

### 9.3 畫面風格規則

- 與官網色溫要協調
- 若產品本身色彩偏硬，官網需用外層容器柔化
- 但不能把產品真實感修飾到失真

## 10. 圖像與裝飾元素規則

如果首頁需要裝飾元素，應使用：

- 柔和幾何塊面
- 光感色帶
- 地形感線條
- 極簡的路線語彙

避免：

- 太多漂浮 icon
- 太多圓點 / 波浪 / 可愛圖案
- 太強的 3D 裝飾
- 太多動態感筆刷

## 11. 文案與視覺組合規則

VELOCUE 最容易出錯的地方，是文字和畫面各做各的。

因此規則是：

- 主標負責價值主張
- 副標負責方法與情境
- 視覺負責證據與氣氛
- CTA 負責把理解收斂成行動

四者不可互相替代。

## 12. 可延伸性規則

所有現在做的首頁視覺語言，都要能延伸到：

- 功能介紹頁
- 更新日誌 / 未來簡介頁
- FAQ 或說明型內容頁

因此不能依賴：

- 只適用首頁的大型主視覺 gimmick
- 太特殊的單次 campaign 版型
- 過度仰賴某一張情境圖

## 13. 設計評估清單

每次做 VELOCUE 視覺稿時，都檢查：

1. 第一眼看上去，是否同時有品牌感與工具感？
2. 暖度是否來自背景與氛圍，而不是色彩堆疊？
3. 大標是否圓潤但仍有份量？
4. 區塊之間是否靠留白建立秩序，而不是靠很多框線？
5. CTA 是否清楚但不促銷？
6. 產品證據是否夠早出現？
7. 這個版型是否能套用到其他頁面？
