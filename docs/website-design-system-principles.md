# VELOCUE 官網設計系統核心原則 v1

最後更新：2026-03-19
適用範圍：VELOCUE 官網、產品介紹頁、首頁、後續品牌型 landing pages
前提文件：

- `docs/website-brand-foundation.md`
- `docs/website-brand-voice-guidelines.md`

## 1. 這份文件的目的

這份文件不是直接規定每個按鈕長什麼樣。
它先定義 VELOCUE 官網設計系統的上位原則，讓後面的版型、色彩、字體、元件、截圖呈現，都有一致的判準。

目標有三個：

- 把審美偏好轉成規則
- 讓官網看起來有品牌質感，但不脫離工具網站本質
- 建立一套能長期延伸的視覺邏輯，而不是一次性的首頁風格

## 2. 設計定位一句話

VELOCUE 的官網設計應該呈現：

`安靜、溫暖、寬鬆、成熟的工具感`

它不是 dashboard 美學，也不是 lifestyle 形象頁。
它是一個實用工具，但表達方式足夠從容、好看、可信。

## 3. 核心設計命題

### 命題 1：工具是本體，品牌感是包裝方式

網站最終仍然要讓人理解 VELOCUE 是可用的工具。

所以設計不能只顧：

- 大圖氛圍
- 感性排版
- 品牌式留白

也要能清楚承接：

- 地圖
- 等高線
- 路線註記
- cue 輸出
- 功能節點說明

設計的任務不是把工具藏起來，而是把工具整理得更值得信任。

### 命題 2：留白要大，但資訊秩序要更強

你偏好大面積留白，這很適合 VELOCUE。
但留白不是空，而是用來建立沉著感與閱讀節奏。

因此 VELOCUE 的留白應該帶來：

- 呼吸感
- 主次清楚
- 低壓閱讀
- 對功能區塊的聚焦

而不是：

- 空洞
- 內容稀薄
- 看起來像尚未填滿的模板

### 命題 3：圓潤可以有，但要帶結構

rounded bold display 是這個品牌的重要識別方向。
但它只能出現在對的位置。

VELOCUE 的圓潤感應該是：

- 穩
- 厚實
- 友善
- 現代

不是：

- 可愛
- 鬆垮
- 玩具感
- 泡泡字感

所以後面在字體、圓角、圖形、按鈕、卡片設計上，都必須遵守：

`rounded, but disciplined`

### 命題 4：暖色調是情緒背景，不是功能主色濫用

你喜歡暖暖的陽光配色，這很適合做品牌氣質。
但暖色如果直接大量進入功能層，容易造成：

- 對比不足
- CTA 不夠穩
- 資訊層級變混
- 看起來太像 lifestyle / cafe / 雜誌專題

因此暖色在 VELOCUE 應主要扮演：

- 品牌底色
- 區塊氛圍
- 插畫 / 背景 / 光感
- hero 和 section 的情緒鋪陳

而功能層仍應保有較清楚的中性色與少量高辨識 accent。

### 命題 5：品牌敘事與產品證據要交錯，不要分裂

VELOCUE 官網不是單純賣感覺。
它也不能只是一頁功能清單。

所以頁面應該同時存在兩種元素：

- 品牌敘事：讓人感受到準備品質、沉著感、上路前的整理感
- 產品證據：讓人看到地圖、等高線、註記、輸出形式真的有被做出來

正確狀態是交錯出現，而不是：

- 上半頁全是品牌，下半頁突然全是規格
- 或整頁從頭到尾都像產品說明書

## 4. VELOCUE 官網應該像什麼

### 應該像

- 一個懂騎乘情境的工具網站
- 一個有審美紀律的品牌頁
- 一個讓人覺得內容經過整理的產品介紹頁
- 一個適合慢慢閱讀，但不拖泥帶水的首頁

### 不應該像

- SaaS 後台模板官網
- 運動科技品牌宣傳頁
- 極簡 lifestyle 品牌型錄
- 太多發光、漸層、玻璃感的 startup landing page
- 資料過滿的功能面板展示頁

## 5. 視覺系統的五個核心原則

### 原則 A：Sunlit Calm

整體視覺應該帶有日光感、空氣感、乾淨感。

關鍵感受：

- 晴朗
- 溫暖
- 沉著
- 乾淨

實作導向：

- 背景不使用純白直鋪到底
- 可使用偏奶白、暖灰白、淡米色作為大底
- 暖色出現在大區塊與微妙層次，不要每個元件都上色
- 陰影要柔和，不要厚重漂浮

### 原則 B：Spacious Precision

留白很多，但排版必須有精準節奏。

實作導向：

- section 與 section 間距要明顯大
- 文字區塊寬度要被控制，不可攤滿
- 一個視覺區塊內的對齊要很穩
- 不靠大量分隔線維持秩序
- 優先用間距，而不是裝飾，建立層級

### 原則 C：Soft Structure

介面語言可以柔和，但骨架必須清楚。

實作導向：

- 圓角偏大，但應規則一致
- 邊框可見但不生硬
- 卡片不要太碎，不要每個資訊都裝進獨立小盒
- 大區塊可以是柔和容器，小元素則盡量克制

### 原則 D：Editorial Utility

首頁的視覺節奏應有 editorial 感，但資訊要能落回工具。

實作導向：

- Hero 可以有較強的字體存在感
- 功能區塊要搭配實際介面證據
- 文字段落需有呼吸，不寫成規格表
- 但每個 section 都應該回答一個明確產品問題

### 原則 E：Measured Accent

強調色要少而準，不可灑滿整頁。

實作導向：

- 主要 CTA 色只保留一到兩種
- 不同功能狀態色不能直接拿來做品牌主視覺
- 頁面不應同時競爭太多亮色
- 高亮只用在真正需要注意的位置

## 6. 色彩策略原則

這一階段先定角色，不先定最終色碼。

### 色彩角色分工

1. 品牌背景色
用來建立暖陽、留白、空氣感。
應偏柔和、低飽和、淺明度。

2. 主文字色
應穩定、成熟、略帶暖度，不能死黑。

3. 結構中性色
用於邊框、次要背景、分層容器、輔助訊息。

4. 主要行動色
用於 CTA、互動重點、關鍵連結。
應乾淨、有信心，不必太暖。

5. 情境輔助色
用於區塊點綴、數據圖像、插畫、提示性差異。
應節制使用。

### 色彩原則

- 暖色調是氣氛主軸，不是整頁主角
- 中性色的品質決定整體成熟度
- 主 CTA 不一定要用最暖的顏色
- 若未來圖表沿用功能色，品牌頁不應直接複製那些圖表色當主視覺
- 對比必須優先於情緒色偏好

## 7. 字體策略原則

這一階段先定角色與規則，不先定最終字體名稱。

### 字體分工

1. Display 字體
用於 hero、大標、短句主張。
應具圓潤、厚實、現代感。

2. Text 字體
用於內文、功能說明、較長段落。
應清楚、穩定、耐讀，避免過度個性。

3. UI 字體
若有需要，可與 text 共用，但在數字、標籤、介面按鈕上要更俐落。

### 字體原則

- Display 可以圓，但 text 不應過度圓
- 大標的個性不能外溢到整頁每一個層級
- 中文字體在粗體下要檢查是否過於可愛
- 英數混排需注意數字與單位在工具語境中的穩定感
- 行高要寬鬆，但不能散

## 8. 版面原則

### 版面節奏

- Hero 寬敞，第一印象要先建立氣質
- 中段開始逐步增加產品證據密度
- 越接近 CTA，越清楚收斂成可行動理解

### 版面結構

- 優先使用大區塊切分，而不是很多小模組拼貼
- 一個 section 以一個主論點為原則
- 大圖或產品視覺旁，必須搭配足夠精準的文字錨點
- 避免滿版三欄 feature 卡一路排到底

### 響應式思維

- 手機版也要保有留白與沉著感，不能因壓縮而變成擁擠資訊流
- 大標在小尺寸下仍需保有穩定節奏
- 圖像與產品示意在手機上應保持「看得懂」，而不只是縮小

## 9. 元件語言原則

這一階段先定語言，不先畫所有 component。

### 按鈕

- 應該穩、簡潔、有重量
- 不要太像電商促銷按鈕
- 不要用過度膨脹的光暈效果

### 卡片

- 應服務內容整理，不應只是裝飾
- 卡片層級數量要少
- 若卡片很多，必須靠版面節奏避免碎裂感

### Section 容器

- 可略帶背景差異或柔和底色
- 用來承接主題切換，而不是每段都做成大盒子

### 標籤 / 狀態 / 小資訊塊

- 功能上可俐落一點
- 不要每個小標籤都做得太可愛
- 以可讀性與秩序優先

## 10. 產品截圖與示意圖原則

VELOCUE 後續首頁很可能需要依賴產品畫面作為可信度來源。

因此產品截圖不應只是「把畫面丟上去」。

原則如下：

- 截圖應該有敘事目的
- 一張圖只證明一件事
- 先讓人看懂地圖 / 等高線 / 註記 / 輸出之間的關係
- 不要同時展示太多細節
- 必要時可做簡化示意，而不是死貼完整 UI

## 11. 插圖 / 攝影 / 圖像風格原則

若未來使用圖像素材，需遵守以下方向：

- 不走熱血賽事主視覺海報風
- 不走過度戶外冒險 rugged 風
- 不走文青生活風雜誌感

比較適合的方向是：

- 明亮自然光
- 有地形、道路、距離感
- 克制的人物存在
- 更像「準備出發前的秩序感」，而不是「衝線瞬間的激情」

## 12. 與產品 UI 的關係

官網設計系統不等於產品 UI 設計系統，但兩者必須同語系。

### 必須共享的東西

- 品牌色溫
- 字體個性方向
- 圓角與幾何語言
- 克制、沉著、可讀的基調

### 必須分開的東西

- 官網可以更有留白、更有敘事節奏
- 產品 UI 必須更高效率、更高資訊密度
- 官網可用較強 display 字感
- 產品 UI 的控制元件必須更直接、更少情緒化

## 13. 設計審查檢查清單

每次做官網設計時，都應檢查：

1. 這個畫面看起來還像工具網站嗎？
2. 它是否過度靠品牌氣氛在撐，而缺乏產品證據？
3. 留白是否有助於理解，而不是只是空？
4. 圓潤感是否開始走向可愛或鬆散？
5. 暖色是否干擾了層級與對比？
6. 元件是否太碎、太多盒、太像模板？
7. 首頁節奏是否在敘事與證據之間平衡？
8. 這個設計是否能延伸到後續頁面，而不只是首頁漂亮？

## 14. 下一階段要落地的內容

這份原則文件下一步會往更具體的方向展開成：

- 首頁資訊架構
- 首頁敘事節奏
- 視覺方向板與風格收斂
- 色彩角色實際定義
- 字體組合提案
- 元件語言與 section 模板

目前先鎖定的總結是：

`VELOCUE 的官網應該是有品牌質感的實用工具頁，而不是看起來漂亮的工具概念圖。`
